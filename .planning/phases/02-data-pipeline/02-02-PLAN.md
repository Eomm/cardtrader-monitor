<task type="auto">
  <name>Task 2: Implement import-wishlist Edge Function</name>
  <files>supabase/functions/import-wishlist/index.ts</files>
  <action>
    Create supabase/functions/import-wishlist/index.ts implementing the full import flow:

    1. Deno.serve handler with CORS support (OPTIONS preflight via corsHeaders)

    2. Auth: Extract Authorization header, create user-scoped Supabase client (anon key + user JWT) for RLS operations, create admin client (service_role key) for token decryption and cache writes

    3. Parse request body: { wishlistUrl: string }
       - Extract wishlist ID from URL using regex: /cardtrader\.com\/(?:\w{2}\/)?wishlists\/(\d+)/
       - Return 400 if invalid URL

    4. Decrypt user's CardTrader API token:
       - Get user ID from supabaseUser.auth.getUser()
       - Call supabaseAdmin.rpc('decrypt_api_token', { target_user_id: userId })
       - Return 400 if no token found ("Please add your CardTrader API token in Settings")

    5. Import flow (following garcon call order):
       a. Fetch expansions via CardTrader API -> build code-to-expansion map
       b. Cache expansions in ct_expansions table (upsert via admin client)
       c. Fetch wishlist items via CardTrader API
       d. Group wishlist items by expansion_code, resolve to expansion_id
       e. For each unique expansion: fetch blueprints -> cache in ct_blueprints (upsert via admin client) -> build collector_number-to-blueprint map
       f. For each wishlist item: resolve to blueprint using expansion + collector_number. If not found, add to skipped list with reason "Blueprint not found"
       g. For each resolved card: fetch marketplace products (rate-limited batches of 8, 1s delay). Apply CT Zero filter. Find cheapest price (null if no offers).

    6. Database inserts (use admin client to bypass RLS for batch efficiency):
       a. Upsert wishlist row in wishlists table (user_id, cardtrader_wishlist_id, name from URL or "Imported Wishlist", last_synced_at = now())
       b. For each resolved card: insert into monitored_cards with:
          - wishlist_id from step a
          - blueprint_id from resolved blueprint
          - card_name = blueprint.name
          - expansion_name from expansion
          - game_id from blueprint
          - collector_number from blueprint.fixed_properties
          - image_url from blueprint
          - baseline_price_cents = cheapest price (null if no offers)
          - notification_rule = { type: 'threshold', threshold_percent: 20, direction: 'both', enabled: true }
          - only_zero = true (default)
          - condition_required = wishlist item condition (or null)
          - language_required = wishlist item language
          - foil_required = normalize wishlist foil string to boolean (foil === "true")
          - is_active = true
       c. If card already exists (same wishlist_id + blueprint_id), skip or update (use ON CONFLICT if possible, or check before insert)

    7. Insert initial price snapshot for each imported card that has a price (price_cents = baseline_price_cents)

    8. Return JSON response:
       ```json
       {
         "imported": 10,
         "skipped": 2,
         "wishlist_id": "uuid-of-wishlist-row",
         "details": [
           { "card_name": "Lightning Bolt", "status": "imported" },
           { "card_name": "Unknown Card", "status": "skipped", "reason": "Blueprint not found" }
         ]
       }
       ```

    9. Error handling:
       - Wrap entire flow in try/catch
       - On CardTrader API errors: return 502 with { error: "CardTrader API error: ..." }
       - On Supabase errors: return 500 with { error: "Database error: ..." }
       - On validation errors (bad URL, no token): return 400

    IMPORTANT: Use Deno env vars (Deno.env.get) for SUPABASE_URL, SUPABASE_ANON_KEY, SUPABASE_SERVICE_ROLE_KEY. These are pre-populated by Supabase in Edge Functions.
  </action>
