    Create supabase/migrations/00002_data_pipeline.sql with:

    1. ALTER monitored_cards: add baseline_price_cents int (nullable — null means no offers found), add retention_days int NOT NULL DEFAULT 30

    2. Create ct_expansions cache table:
       - id int PRIMARY KEY
       - game_id int NOT NULL
       - code text NOT NULL UNIQUE
       - name text NOT NULL
       - fetched_at timestamptz NOT NULL DEFAULT now()

    3. Create ct_blueprints cache table:
       - id int PRIMARY KEY
       - expansion_id int NOT NULL REFERENCES ct_expansions(id)
       - name text NOT NULL
       - game_id int
       - collector_number text
       - image_url text
       - scryfall_id text
       - fetched_at timestamptz NOT NULL DEFAULT now()
       - Indexes: on expansion_id, on (expansion_id, collector_number)

    4. Create decrypt_api_token(target_user_id uuid) RETURNS text function:
       - SECURITY DEFINER
       - Uses pgp_sym_decrypt with current_setting('app.settings.encryption_key')
       - Queries profiles table by target_user_id (NOT auth.uid() — this is for Edge Function/service role use)
       - Returns null if no token found

    5. RLS for cache tables:
       - Enable RLS on both ct_expansions and ct_blueprints
       - SELECT policy: allow all (public reference data)
       - INSERT/UPDATE policies: only service_role (no check needed — these are populated by Edge Functions and GH Actions which use service_role)

    6. Add INSERT policy for price_snapshots (currently only SELECT exists):
       - price_snapshots_insert: allow insert when monitored_card belongs to user's wishlist (same pattern as monitored_cards_insert)
       - Also add a service-role-friendly version: the GH Actions job uses service_role which bypasses RLS, so no special policy needed for that case
