  <name>Task 1: Implement price fetch script</name>
  <files>scripts/fetch-prices.ts</files>
  <action>
    Create scripts/fetch-prices.ts -- a standalone Node.js/TypeScript script that:

    IMPORTANT: Structure the script with a main() guard pattern. Export an async `main()` function containing all logic, and call it at module scope only via:
    ```typescript
    export async function main() { /* all logic here */ }
    main().catch((err) => { console.error('Fatal:', err); process.exit(1); });
    ```
    This ensures the module can be imported for validation without executing side effects, and that env var errors at module scope don't get silently swallowed.

    1. Inside main(): Read env vars: SUPABASE_URL, SUPABASE_SERVICE_ROLE_KEY, ENCRYPTION_KEY
       - Validate all are present, exit with error if missing
       - Create Supabase client with service_role key (bypasses RLS)

    2. Query all active monitored cards:
       ```sql
       SELECT mc.*, w.user_id
       FROM monitored_cards mc
       JOIN wishlists w ON w.id = mc.wishlist_id
       WHERE mc.is_active = true
       ```
       - Exit early with "No active cards to monitor" if empty

    3. Deduplicate by blueprint_id:
       - Group cards by blueprint_id
       - For each unique blueprint_id, pick one representative card's filters (they should be the same for same blueprint)
       - Log: "Found {N} active cards across {M} unique blueprints"

    4. Get API tokens:
       - Query distinct user_ids from the active cards
       - For each user, call decrypt_api_token RPC to get their CardTrader token
       - Use the first available valid token for marketplace queries (per research recommendation -- marketplace prices are the same regardless of querying user)
       - If no valid tokens found, exit with error

    5. Fetch prices in rate-limited batches:
       - Import and use processBatches from src/lib/cardtrader-utils.ts (the script runs in Node.js with tsx, so it can import from src/)
       - For each unique blueprint_id: fetch GET /marketplace/products?blueprint_id={id}&language={lang}
       - On individual fetch failure: log error, continue with next blueprint (partial data > no data)
       - Apply CT Zero filter using filterCtZeroOffers
       - Find cheapest price using findCheapestPrice

    6. Insert price snapshots:
       - For each monitored card: insert into price_snapshots with price_cents = cheapest price for its blueprint
       - If no offers found (cheapest is null): insert with price_cents = null? No -- price_cents is NOT NULL in schema. Skip snapshot for cards with no offers. Log "No offers for blueprint {id}"
       - Batch insert snapshots for efficiency (supabase .insert() with array)
       - Include marketplace_data jsonb with top 3 offers for reference

    7. Log summary:
       - "Price fetch complete: {N} snapshots created for {M} cards ({K} blueprints queried, {F} failed)"
       - Total execution time

    IMPORTANT: Import from src/lib/ is valid because tsx resolves TypeScript imports. Use relative paths: `import { processBatches, filterCtZeroOffers, findCheapestPrice, deduplicateBlueprintIds } from '../src/lib/cardtrader-utils'`

    IMPORTANT: The CardTrader marketplace API base URL is https://api.cardtrader.com/api/v2. Auth header: `Authorization: Bearer {token}`. Response for /marketplace/products is an object keyed by blueprint_id (string), NOT an array.
  </action>

    on:
      schedule:
        - cron: '0 * * * *'  # Every hour at minute 0
      workflow_dispatch: {}    # Manual trigger for testing

    jobs:
      fetch:
        runs-on: ubuntu-latest
        steps:
          - uses: actions/checkout@v5
          - uses: actions/setup-node@v6

              node-version: lts/*
              cache: 'npm'
          - run: npm ci
          - run: npx tsx scripts/fetch-prices.ts

              SUPABASE_URL: ${{ secrets.SUPABASE_URL }}

              ENCRYPTION_KEY: ${{ secrets.ENCRYPTION_KEY }}
