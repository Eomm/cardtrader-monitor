Create docs/SETUP.md with the following sections. Claude's discretion on exact format and structure.

**Prerequisites:**
- Node.js (version from package.json engines or .nvmrc if exists, otherwise latest LTS)
- Supabase account (free tier)
- CardTrader account with API token
- Telegram Bot (created via @BotFather)
- GitHub account (for GitHub Pages deployment and Actions)

**Local Development Setup:**
1. Clone the repo
2. `npm install`
3. Copy `.env.example` to `.env` (or create .env with required vars)
4. List all required environment variables:
   - `VITE_SUPABASE_URL` -- Supabase project URL
   - `VITE_SUPABASE_ANON_KEY` -- Supabase anon key
   - `SUPABASE_URL` -- same URL for scripts
   - `SUPABASE_SERVICE_ROLE_KEY` -- service role key for scripts/Edge Functions
   - `ENCRYPTION_KEY` -- for CardTrader token encryption
   - `TELEGRAM_BOT_TOKEN` -- Telegram bot token
5. `npm run dev` to start local dev server

**Supabase Setup:**
1. Create a new Supabase project
2. Run migrations: `npx supabase db push` (or manual SQL in dashboard)
3. Deploy Edge Functions: `npm run supabase:deploy` (document the actual command)
4. **Important:** Set Edge Functions to NOT verify JWT with legacy secret (per project memory)

**Google OAuth Configuration:**
1. Go to Google Cloud Console -> APIs & Credentials
2. Create OAuth 2.0 Client ID (Web application)
3. Add authorized redirect URI: `https://{supabase-project-ref}.supabase.co/auth/v1/callback`
4. Copy Client ID and Client Secret
5. In Supabase Dashboard -> Authentication -> Providers -> Google: enable and paste credentials
6. Set Site URL to your deployed URL (e.g., `https://username.github.io/cardtrader-monitor`)
7. Add redirect URLs for local dev (`http://localhost:5173/cardtrader-monitor/dashboard`)

**Telegram Bot Setup:**
1. Message @BotFather on Telegram, send /newbot
2. Copy the bot token to `TELEGRAM_BOT_TOKEN` env var
3. Set webhook URL: document the Supabase Edge Function URL for the telegram webhook

**GitHub Actions Setup:**
1. Add repository secrets: `SUPABASE_URL`, `SUPABASE_SERVICE_ROLE_KEY`, `ENCRYPTION_KEY`, `TELEGRAM_BOT_TOKEN`
2. Enable GitHub Pages (Settings -> Pages -> Source: GitHub Actions)
3. Workflows: CI (biome check), Deploy (GitHub Pages), Fetch Prices (hourly), Sync Wishlists (daily), Cleanup (daily)

**Available Scripts:**
