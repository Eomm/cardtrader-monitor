  <files>src/index.css, src/App.tsx, src/components/Navbar.tsx, src/components/CardList.tsx, src/components/CardRow.tsx, src/components/ImportWishlistForm.tsx, src/components/PriceDisplay.tsx, src/components/ProtectedRoute.tsx, src/components/RuleEditor.tsx, src/pages/LoginPage.tsx, src/pages/DashboardPage.tsx, src/pages/SettingsPage.tsx, src/pages/CardDetailPage.tsx</files>
  <action>
**Step 1: Remove @theme block from src/index.css.** Delete the entire `@theme { ... }` block. Keep only `@import "tailwindcss"` and the body styles. Tailwind v4 includes all color scales (slate, blue, emerald, red) by default.

**Step 2: Apply color mapping across ALL 12+ .tsx files.** Read each file and replace colors using this mapping:

| Old Pattern | New Pattern |
|-------------|-------------|
| `bg-papaya dark:bg-deep-space` | `bg-slate-900` |
| `bg-deep-space` | `bg-slate-900` |
| `bg-papaya` | `bg-slate-100` only if used for light surface, otherwise `bg-slate-800` |
| `bg-white dark:bg-deep-space/50` | `bg-slate-800` |
| `bg-white dark:bg-deep-space` | `bg-slate-800` |
| `text-papaya` | `text-slate-100` |
| `text-deep-space` | `text-slate-100` (on dark bg) |
| `text-deep-space/60`, `text-papaya/60` | `text-slate-400` |
| `text-deep-space/50`, `text-papaya/50` | `text-slate-500` |
| `bg-steel` | `bg-blue-500` |
| `text-steel` | `text-blue-500` |
| `hover:bg-steel/80` | `hover:bg-blue-600` |
| `border-steel/20`, `border-steel` | `border-slate-700` |
| `bg-flag-red`, `text-flag-red` | `bg-red-500`, `text-red-500` |
| `text-molten` | `text-red-500` |
| `bg-steel/10` | `bg-slate-700` |
| `ring-steel` | `ring-blue-500` |

**Step 3: Remove ALL `dark:` prefixes.** The app is now dark-only. Every `dark:X` becomes just `X`, and the light-mode counterpart is removed. For example, `bg-white dark:bg-slate-800` becomes `bg-slate-800`.

**Step 4: Clean up remaining opacity modifiers.** Replace `text-X/60` patterns with specific slate shades instead (slate-400 for muted, slate-500 for dimmer).

**Step 5: Update body/root background.** In App.tsx AuthenticatedLayout, ensure the outer div uses `bg-slate-900 text-slate-100`. In LoginPage, ensure the same dark background.

**Step 6: Verify no old color names remain.** Grep for `deep-space|papaya|flag-red|steel|molten` across all src/ files. Zero hits expected.

**Step 7: Verify no `dark:` prefixes remain.** Grep for `dark:` in src/. Zero hits expected.

**Pitfall:** Do NOT use opacity modifiers like `text-slate-100/60` -- use specific shades like `slate-400` instead. The whole point of moving to a full color scale is having granular shades.

Run `npm run build` to verify the build succeeds.
