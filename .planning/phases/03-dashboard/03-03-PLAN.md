2. Create `src/pages/CardDetailPage.tsx`:
   - Use `useParams<{ id: string }>()` from react-router to get card ID
   - Fetch single card: `supabase.from('monitored_cards').select('*').eq('id', id).single()`
   - Fetch latest price: `supabase.from('price_snapshots').select('price_cents').eq('monitored_card_id', id).order('recorded_at', { ascending: false }).limit(1).single()`
   - Build `MonitoredCardWithPrice` by merging card data with latest price
   - Layout (per user decision): card image on the left, info sections stacked on the right
   - Mobile responsive: flex-col on small screens, flex-row on md+
   - Image: larger than list thumbnail, object-contain, with placeholder if null

   **Section 1 - Price:**
   - Current price (using `formatEur` from PriceDisplay.tsx), clickable -- opens `https://www.cardtrader.com/it/cards/${card.blueprint_id}` in new tab (`window.open` or `<a target="_blank" rel="noopener noreferrer">`)
   - Baseline price
   - Price change percentage (use `PriceChange` component)

   **Section 2 - Card Properties:**
   - Condition: display `card.condition_required` or "Any" if null (read-only)
   - Language: display flag emoji via `languageToFlag(card.language_required)` + language code (read-only)
   - Foil: display "Yes"/"No"/"Any" based on `card.foil_required` (read-only, null = Any)
   - CT Zero: toggle switch (editable). When toggled, update `only_zero` in Supabase immediately (this is a simple boolean, no save button needed -- it's a filter preference, not a rule)
   - Expansion: `card.expansion_name`

   **Section 3 - Rule Editor:**
   - Render `<RuleEditor cardId={card.id} rules={card.notification_rule ?? []} onSave={handleRulesSaved} />`
   - `handleRulesSaved` refreshes card data

   **Stop Monitoring button:**
   - Red-toned button at the bottom: "Stop Monitoring"
   - On click: confirm with inline confirmation pattern (per project convention from 01-02 -- inline confirmation, not browser confirm dialog)
   - Sets `is_active = false` via `supabase.from('monitored_cards').update({ is_active: false }).eq('id', cardId)`
   - After success, navigate back to dashboard
   - If card is already inactive, show "Resume Monitoring" button instead (sets `is_active = true`)

   **States:**
   - Loading: spinner (same pattern as DashboardPage)
   - Error: error message with retry button
   - Not found: "Card not found" message

3. Create `src/components/RuleEditor.tsx`:
   - Props: `{ cardId: string; rules: NotificationRule[]; onSave: () => void }`
   - Local state: `useState<NotificationRule[]>(rules)` for form state
   - Two tabs/sections: "Threshold" and "Stability" (use button tabs, active tab highlighted with steel bg)

   **Threshold tab:**
   - List existing threshold rules from the array
   - "Add Threshold Rule" button if none exist
   - Each threshold rule shows:
     - Percentage: `<input type="number" min="1" max="100" />` (free number input per user decision)
     - Direction: three-button toggle for "Up" / "Down" / "Both"
     - Enabled: toggle switch
     - Remove button (trash icon or "Remove" text)

   **Stability tab:**
   - List existing stability rules from the array
   - "Add Stability Rule" button if none exist
   - Each stability rule shows:
     - Range %: `<input type="number" min="1" max="100" />`
     - Consecutive days: `<input type="number" min="1" max="30" />`
     - Enabled: toggle switch
     - Remove button

   **Save button:**
   - Explicit save button at the bottom (per user decision -- NOT auto-save)
   - On click: `supabase.from('monitored_cards').update({ notification_rule: localRules }).eq('id', cardId)`
   - Show success feedback (brief "Saved!" text or button color change)
   - Call `onSave()` after successful save

   **Default values when adding new rule:**
   - Threshold: use `createDefaultNotificationRule()` (20%, both, enabled)
   - Stability: use `createDefaultStabilityRule()` (5% range, 3 days, enabled)

   **Styling:**
   - Use project theme colors (steel for borders/accents, deep-space/papaya for text, flag-red for destructive actions)
   - Form inputs: consistent with existing input styling in the project (border-steel/20, rounded, px-3 py-2)
