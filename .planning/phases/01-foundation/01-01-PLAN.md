3. Create `supabase/migrations/00001_initial_schema.sql` with the FULL application schema from the design doc:

   **Extensions:**
   - `CREATE EXTENSION IF NOT EXISTS pgcrypto;`

   **Tables (5 total):**
   - `profiles` (id uuid PK refs auth.users, cardtrader_api_token text encrypted, telegram_chat_id bigint, telegram_link_token text, created_at, updated_at)
   - `wishlists` (id uuid PK, user_id refs profiles, cardtrader_wishlist_id text, name text, last_synced_at, created_at, updated_at)
   - `monitored_cards` (id uuid PK, wishlist_id refs wishlists, blueprint_id bigint, card_name, expansion_name, game_id int, collector_number, image_url, notification_rule jsonb, only_zero boolean default true, condition_required, language_required, foil_required boolean, reverse_required boolean, first_edition_required boolean, is_active boolean default true, created_at, updated_at)
   - `price_snapshots` (id uuid PK, monitored_card_id refs monitored_cards, price_cents int, recorded_at, marketplace_data jsonb)
   - `notifications` (id uuid PK, monitored_card_id refs monitored_cards, notification_type text, old_price_cents int, new_price_cents int, sent_at, telegram_message_id text)

   **Indexes (7):**
   - idx_wishlists_user_id, idx_monitored_cards_wishlist_id, idx_monitored_cards_blueprint_id, idx_price_snapshots_card_id, idx_price_snapshots_recorded_at (DESC), idx_notifications_card_id

   **Trigger -- auto-create profile on signup:**
   ```sql
   CREATE OR REPLACE FUNCTION public.handle_new_user()
   RETURNS TRIGGER AS $$
   BEGIN
     INSERT INTO public.profiles (id) VALUES (NEW.id);
     RETURN NEW;
   END;
   $$ LANGUAGE plpgsql SECURITY DEFINER;

   CREATE TRIGGER on_auth_user_created
     AFTER INSERT ON auth.users
     FOR EACH ROW EXECUTE FUNCTION public.handle_new_user();
   ```

   **Token encryption functions (3):**
   - `save_api_token(token text)` -- pgp_sym_encrypt with `current_setting('app.settings.encryption_key')`, SECURITY DEFINER, updates where id = auth.uid()
   - `has_api_token()` -- returns boolean, checks if cardtrader_api_token IS NOT NULL for auth.uid()
   - `remove_api_token()` -- sets cardtrader_api_token to NULL for auth.uid()

   **RLS policies:**
   - Enable RLS on all 5 tables
   - profiles: SELECT/UPDATE own row (id = auth.uid())
   - wishlists: SELECT/INSERT/UPDATE/DELETE own rows (user_id = auth.uid())
   - monitored_cards: SELECT/INSERT/UPDATE/DELETE via wishlist join (wishlist.user_id = auth.uid())
   - price_snapshots: SELECT via monitored_cards join chain
   - notifications: SELECT via monitored_cards join chain

   **Updated_at trigger function:**
   - Generic `set_updated_at()` function
   - Apply to profiles, wishlists, monitored_cards

4. Run biome check to ensure workflow YAML does not break anything.
