  <name>Task 1: Telegram Edge Functions (webhook + test message)</name>
  <files>supabase/functions/_shared/telegram.ts, supabase/functions/telegram-webhook/index.ts, supabase/functions/send-test-message/index.ts</files>
  <action>
    **1. Create `supabase/functions/_shared/telegram.ts`:**
    Shared Telegram helper for Deno Edge Functions (re-implemented for Deno runtime, same pattern as cardtrader-api.ts being separate from src/lib/).

    ```typescript
    export async function sendTelegramMessage(
      botToken: string,
      chatId: string | number,
      text: string,
      parseMode: 'MarkdownV2' | 'HTML' = 'MarkdownV2',
    ): Promise<{ ok: boolean; messageId?: string; error?: string }>
    ```

    - POST to `https://api.telegram.org/bot{token}/sendMessage`
    - Body: `{ chat_id, text, parse_mode, link_preview_options: { is_disabled: true } }`
    - Return `{ ok: true, messageId }` on success, `{ ok: false, error }` on failure

    **2. Create `supabase/functions/telegram-webhook/index.ts`:**
    Public endpoint (no JWT verification) that Telegram calls when users message the bot.

    - Only handle POST requests (return 200 OK for others)
    - Parse the webhook update JSON
    - If message text starts with `/start`: reply with the user's chat ID in a MarkdownV2 message
    - Reply text: "Your chat ID is: `{chatId}`\n\nCopy this number and paste it in the CardTrader Monitor settings page to connect notifications\."
    - Escape the instructional text properly for MarkdownV2 (periods, parentheses etc.)
    - Use `Deno.env.get('TELEGRAM_BOT_TOKEN') ?? ''` for bot token (Biome-compliant)
    - Always return 200 OK to Telegram (even on errors, to prevent retries)

    **3. Create `supabase/functions/send-test-message/index.ts`:**
    Authenticated endpoint (uses JWT) called from Settings page to verify Telegram connection.

    - Handle CORS preflight with shared handleCors helper
    - Extract user from Supabase auth (same pattern as import-wishlist)
    - Read `chat_id` from request body
    - Send a test message: "CardTrader Monitor connected successfully\! You will receive price alerts here\."
    - Use plain text (no MarkdownV2 needed for test message, or escape properly if using MarkdownV2)
    - Return JSON `{ ok: true }` or `{ ok: false, error: "..." }`
    - Do NOT set `Verify JWT with legacy secret` to false for this function (it requires auth)
  </action>
