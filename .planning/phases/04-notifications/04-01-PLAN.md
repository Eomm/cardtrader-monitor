    ## Threshold Evaluation (tests/threshold-evaluation.test.ts)
    - evaluateThreshold(rule, baselineCents=1000, currentCents=800) with threshold_percent=15, direction='down' -> { triggered: true, percentChange: -20 }
    - evaluateThreshold(rule, baselineCents=1000, currentCents=800) with threshold_percent=25, direction='down' -> { triggered: false, percentChange: -20 } (20% < 25% threshold)
    - evaluateThreshold(rule, baselineCents=1000, currentCents=1200) with direction='down' -> { triggered: false, percentChange: 20 } (price rose but direction is down-only)
    - evaluateThreshold(rule, baselineCents=1000, currentCents=1200) with direction='up' -> { triggered: true, percentChange: 20 }
    - evaluateThreshold(rule, baselineCents=1000, currentCents=800) with direction='both' -> { triggered: true, percentChange: -20 }
    - evaluateThreshold(rule, baselineCents=null, currentCents=500) -> { triggered: true, percentChange: -100 } (null baseline = infinity, any price is a drop)
    - evaluateThreshold(rule, baselineCents=null, currentCents=500) with direction='up' -> { triggered: false, percentChange: -100 } (null baseline drop, but rule is up-only)
    - evaluateThreshold(rule, baselineCents=1000, currentCents=null) -> { triggered: false, percentChange: 0 } (no offers = skip)
    - evaluateThreshold(disabledRule, baselineCents=1000, currentCents=500) -> { triggered: false, percentChange: 0 } (disabled rules skipped)

    ## Cooldown Logic (tests/cooldown.test.ts)
    - shouldNotify(card, currentPrice, null) -> uses baseline, no cooldown
    - shouldNotify(card, currentPrice, { price: 900, sentAt: 23h ago }) -> uses last notified price (900) as baseline instead of original baseline
    - shouldNotify(card, currentPrice, { price: 900, sentAt: 25h ago }) -> cooldown expired, uses original baseline
    - shouldNotify during cooldown where price moved enough from last alert price -> bypass cooldown, return triggered

    ## Message Formatting (tests/notification-message.test.ts)
    - escapeMarkdownV2("Hello_World") -> "Hello\\_World"
    - escapeMarkdownV2("Price: €15.00") -> "Price: €15\\.00" (dot escaped)
    - escapeMarkdownV2Url("https://example.com/path)more") -> "https://example.com/path\\)more"
    - formatAlertMessage with one alert -> single line: emoji + [card name](url) + old price -> new price (pct%)
    - formatAlertMessage with price drop -> green circle emoji
    - formatAlertMessage with price rise -> red circle emoji
    - formatAlertMessage with multiple alerts -> newline-separated lines
    - formatEurCents(1727) -> "€17.27"

  <implementation>
    Create `src/lib/telegram-utils.ts` with these exports:

    1. **escapeMarkdownV2(text: string): string** — escape all 18 MarkdownV2 special characters: `_*[]()~\`>#+-=|{}.!\\`

    2. **escapeMarkdownV2Url(url: string): string** — escape only `)` and `\\` in URLs

    3. **formatEurCents(cents: number): string** — format cents as `€XX.XX`

    4. **ThresholdAlert interface:**
       ```typescript
       interface ThresholdAlert {
         cardName: string;
         blueprintId: number;
         oldPriceCents: number;
         newPriceCents: number;
         percentChange: number;
       }
       ```

    5. **evaluateThreshold(rule: ThresholdRule, baselineCents: number | null, currentCents: number | null): { triggered: boolean; percentChange: number }**
       - currentCents null -> skip (no offers)
       - Disabled rule -> skip
       - baselineCents null -> treat as infinity, percentChange = -100, triggered if direction allows drops
       - Otherwise: compute percentChange = ((current - baseline) / baseline) * 100
       - Check abs(percentChange) >= threshold_percent
       - Check direction match (down: percentChange < 0, up: percentChange > 0, both: either)

    6. **shouldNotify(rule: ThresholdRule, baselineCents: number | null, currentCents: number | null, lastNotification: { priceCents: number; sentAt: Date } | null): { triggered: boolean; percentChange: number; comparisonPriceCents: number | null }**
       - If no lastNotification or lastNotification older than 24h: evaluate against baseline
       - If within 24h cooldown: evaluate against lastNotification.priceCents instead of baseline
       - Returns which price was used as comparison (for logging)

    7. **formatAlertMessage(alerts: ThresholdAlert[]): string**
       - Per user decision format: `emoji [cardName](cardtraderUrl) oldPrice \-> newPrice \(pct%\)`
       - Green circle for drops (percentChange < 0), red circle for rises
       - URL: `https://www.cardtrader.com/en/cards/{blueprintId}`
       - Join with newlines
       - If total length > 4000 chars, split into chunks (return array? or truncate with "and N more...")
  </implementation>
