    - `baseline_price_cents: number | null`
    - `notification_rule: unknown` (will be parsed as NotificationRule[])

    **Update the Supabase query** (step 2) to also select:
    - `card_name, baseline_price_cents, notification_rule` from monitored_cards
    - `wishlists!inner(user_id, profiles!inner(telegram_chat_id))` -- join through to profiles for chat ID

    **Flatten the join** to include `card_name`, `baseline_price_cents`, `notification_rule`, and `telegram_chat_id` on each card row.

    **Add new steps after snapshot insertion (after existing step 6):**

    **Step 7: Read TELEGRAM_BOT_TOKEN from env.** If not set, log "TELEGRAM_BOT_TOKEN not set, skipping notifications" and return (graceful skip -- don't fail the job).

    **Step 8: Query recent notifications (last 24h)** for all active card IDs to build cooldown map:
    ```typescript
    const { data: recentNotifs } = await supabase
      .from('notifications')
      .select('monitored_card_id, new_price_cents, sent_at')
      .in('monitored_card_id', cards.map(c => c.id))
      .gte('sent_at', new Date(Date.now() - 24 * 60 * 60 * 1000).toISOString())
      .order('sent_at', { ascending: false });
    ```
    Build a Map<string, { priceCents: number; sentAt: Date }> with the most recent notification per card.

    **Step 9: Evaluate thresholds for each card.**
    For each card:
    - Parse `notification_rule` as NotificationRule[] (with Array.isArray defensive check)
    - Filter to ThresholdRule entries only (type === 'threshold')
    - Skip disabled rules
    - Get current cheapest price from the snapshots map (built in step 6)
    - Call `shouldNotify(rule, card.baseline_price_cents, currentPriceCents, lastNotifForCard)` from telegram-utils
    - If triggered, add to alerts list grouped by user's telegram_chat_id

    **Step 10: Send Telegram messages.**
    For each user (grouped by telegram_chat_id):
    - Skip if no telegram_chat_id (user hasn't connected Telegram)
    - Call `formatAlertMessage(alerts)` from telegram-utils
    - If message > 4000 chars, split into chunks
    - Send via direct fetch to `https://api.telegram.org/bot{TOKEN}/sendMessage` (same pattern as the Edge Function helper but inline in Node.js -- do NOT import from supabase/functions/_shared/)
    - Implementation: create a local `sendTelegramMessage` function in fetch-prices.ts (Node.js runtime, cannot import Deno code)

    **Step 11: Log notifications to database.**
    For each successfully sent alert, insert into `notifications` table:
    ```typescript
    {
      monitored_card_id: alert.cardId,
      notification_type: 'threshold',
      old_price_cents: alert.oldPriceCents,
      new_price_cents: alert.newPriceCents,
      sent_at: new Date().toISOString(),
      telegram_message_id: result.messageId ?? null,
    }
    ```
    Use a single batch insert for all notifications.

    **Step 12: Update summary log** to include notification count:
    `"Price fetch complete: X snapshots, Y notifications sent for Z cards..."`

    **Important implementation notes:**
    - Import `evaluateThreshold`, `shouldNotify`, `formatAlertMessage`, `ThresholdAlert` from `../src/lib/telegram-utils`
    - Import `ThresholdRule`, `NotificationRule` from `../src/lib/cardtrader-types`
    - The `sendTelegramMessage` function must be defined locally in this file (Node.js, not Deno)
    - Use `link_preview_options: { is_disabled: true }` (not the deprecated `disable_web_page_preview`)
    - Pass chat_id as string to Telegram API to avoid bigint precision issues
    - If Telegram API call fails for a user, log the error but continue with other users (don't fail the entire job)
    - Biome compliance: no non-null assertions, use ?? fallbacks
