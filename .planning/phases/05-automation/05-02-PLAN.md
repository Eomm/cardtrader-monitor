Create `scripts/sync-wishlists.ts` following the fetch-prices.ts pattern. The script does:

**Setup (same pattern as fetch-prices.ts):**
- Import createClient from @supabase/supabase-js
- Import types from src/lib/cardtrader-types.ts (Blueprint, Expansion, MarketplaceProduct, CardFilters)
- Import utilities from src/lib/cardtrader-utils.ts (processBatches, findCheapestPrice, createDefaultNotificationRule, mapBlueprintToCard)
- Define CARDTRADER_API_BASE = 'https://api.cardtrader.com/api/v2'
- Export async function main(), validate SUPABASE_URL + SUPABASE_SERVICE_ROLE_KEY

**CardTrader API helpers (re-implement for Node.js, same as Deno _shared/cardtrader-api.ts but using native fetch):**
- fetchExpansions(token) -> Map<string, Expansion> (keyed by code)
- fetchWishlist(token, wishlistId) -> { id, name, game_id, items: WishlistItem[] }
- fetchBlueprintsForExpansion(token, expansionId) -> Map<string, Blueprint> (keyed by collector_number)
- fetchMarketplaceProducts(token, blueprintId, language) -> MarketplaceProduct[] (response keyed by blueprint_id string)
- Define WishlistItem interface inline: { quantity, meta_name, expansion_code, collector_number, language, condition, foil: string | boolean }

**Main flow:**
1. Query all wishlists: supabase.from('wishlists').select('id, user_id, cardtrader_wishlist_id, name')
2. Group wishlists by user_id using a Map
3. Track totals: totalAdded, totalRemoved, totalUpdated, totalSkippedUsers
4. For each user (sequential iteration):
   a. Wrap in try/catch -- on error, log "Sync failed for user {userId}: {error}" and continue (increment totalSkippedUsers)
   b. Get API token: supabase.rpc('get_api_token', { target_user_id: userId })
   c. If no token, log warning and continue
   d. For each wishlist of this user, call syncWishlist()

**syncWishlist(supabase, apiToken, wishlist) function:**
1. Fetch current wishlist from CardTrader API: fetchWishlist(apiToken, String(wishlist.cardtrader_wishlist_id))
2. Get current DB cards for this wishlist: supabase.from('monitored_cards').select('id, blueprint_id, card_name, image_url, expansion_name').eq('wishlist_id', wishlist.id)
3. Resolve wishlist items to blueprint IDs:
   - Fetch expansions: fetchExpansions(apiToken)
   - Group wishlist items by expansion_code
   - For each unique expansion with new items: check ct_blueprints cache first (supabase.from('ct_blueprints').select('*').eq('expansion_id', expansion.id)), only fetch from API if not cached
   - Cache any newly fetched blueprints to ct_blueprints and ct_expansions tables
   - Build resolvedItems map: blueprint_id -> { item: WishlistItem, blueprint: Blueprint, expansionName: string }
4. Build sets for diff:
   - apiBlueprints: Set of blueprint_ids from resolved items
   - dbCards: Map of blueprint_id -> DB row
5. Determine changes:
   - toAdd: blueprint_ids in apiBlueprints but not in dbCards
   - toRemove: DB card ids where blueprint_id is in dbCards but not in apiBlueprints
   - toUpdate: blueprint_ids in both sets
6. **Add new cards** (for each toAdd):
   - Get resolved item data (blueprint, expansion name)
   - Fetch baseline price: fetchMarketplaceProducts for the blueprint, then findCheapestPrice with filters from the wishlist item (condition, language, foil, onlyZero: true)
   - Use processBatches(toAddItems, ..., 8, 1000) for rate-limited API calls
   - Insert monitored_card: use mapBlueprintToCard() + add wishlist_id, baseline_price_cents, notification_rule: [createDefaultNotificationRule()], only_zero: true, condition_required, language_required, foil_required (from wishlist item), is_active: true
   - Upsert with onConflict: 'wishlist_id,blueprint_id' to handle race conditions
7. **Remove old cards** (for each toRemove):
   - DELETE FROM monitored_cards WHERE id IN (...toRemoveIds)
   - CASCADE handles snapshots and notifications automatically
8. **Update metadata** (for each toUpdate):
   - Only update: card_name, image_url, expansion_name from the resolved blueprint data
   - NEVER touch: notification_rule, baseline_price_cents, only_zero, condition_required, language_required, foil_required, retention_days, is_active
   - Use supabase.from('monitored_cards').update({ card_name, image_url, expansion_name, updated_at: new Date().toISOString() }).eq('id', dbCard.id)
9. Update wishlists.last_synced_at: supabase.from('wishlists').update({ last_synced_at: new Date().toISOString() }).eq('id', wishlist.id)
10. Return { added: toAdd.length, removed: toRemove.length, updated: toUpdate.length }

**Summary log:**
- "Sync complete: {totalAdded} added, {totalRemoved} removed, {totalUpdated} updated, {totalSkippedUsers} users skipped in {elapsed}s"

**Key constraints (from user decisions):**
- Hard-delete (not soft-delete/deactivate) removed cards
- Card removed then re-added = brand new card (fresh baseline, fresh rules)
- Multiple wishlists per user: sync operates per-wishlist, never cross-wishlist delete
- Silent sync -- no Telegram notifications about sync changes
- Keep original baseline price for existing cards (never refresh on sync)
- Update metadata only for card_name, image_url, expansion_name
- If CardTrader API is down: log error and skip that user, continue with others

**Foil normalization:** Use same pattern as import-wishlist: if typeof foil === 'boolean' return foil; else return foil === 'true'
