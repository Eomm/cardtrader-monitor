  <behavior>
    Fixed price evaluation — crossing semantics:
    - direction='down': triggers when currentCents <= targetCents AND (no previous price OR previousCents > targetCents)
    - direction='up': triggers when currentCents >= targetCents AND previousCents < targetCents
    - direction='both': triggers on either crossing direction
    - enabled=false: never triggers regardless of prices
    - currentCents=null: never triggers (no price data)
    - targetCents derived from rule.price_eur * 100 (EUR to cents conversion)

    Default factory:
    - createDefaultFixedPriceRule() returns { type: 'fixed_price', price_eur: 1.00, direction: 'down', enabled: true }

    Integration in fetch-prices.ts:
    - Fixed price rules are filtered alongside threshold rules from the notification_rule array
    - Each enabled fixed_price rule is evaluated per card
    - First triggered rule (threshold OR fixed_price) sends one alert per card (existing dedup behavior)
    - formatAlertMessage reused as-is (it only needs oldPriceCents, newPriceCents, percentChange)
  </behavior>
  <implementation>
    1. Add FixedPriceRule interface to cardtrader-types.ts and extend NotificationRule union

    4. Extend tests/notification-rule.test.ts with createDefaultFixedPriceRule test case
    5. Create tests/fixed-price-evaluation.test.ts with crossing semantics tests
    6. Extend the threshold evaluation loop in scripts/fetch-prices.ts (around line 376) to also filter and evaluate fixed_price rules. Import FixedPriceRule type. After evaluating threshold rules, evaluate fixed_price rules. Maintain the existing "one alert per card" dedup with break.
